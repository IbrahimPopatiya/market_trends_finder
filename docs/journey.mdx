## 🏗️ Phase 1: Self-Evolving Infrastructure
**Date**: April 6, 2026
**Status**: In Progress 🏗️

### Objective
Build a system where the agent is not just a "script runner," but a "skill builder." Every run should contribute tools (utils) and dependencies back to the main engine.

### Key Decisions
1. **Dockerized Sandboxing**: Each workflow run happens in an isolated Docker container.
2. **Post-Run Synchronization**: A sync mechanism detects new scripts in `utils/` and new entries in `requirements.txt`, merging them into the master workspace.
3. **CLI-Based Interface**: Orchestration is handled via a Python-based CLI (`run_workflow.py`).
